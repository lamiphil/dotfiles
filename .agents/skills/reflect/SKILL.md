# Reflection

Review the current session for reusable learnings without polluting durable docs.

## When to Use This Skill

- The user asks to reflect on the current session.
- The user asks what was learned, what to note, or what would have helped the agent move faster.
- The user asks whether project docs, agent instructions, or skills should be updated.

## Core Rules

- Keep the bar high; most sessions produce nothing worth codifying.
- Do not summarize the work; capture only reusable learnings.
- Propose edits only; do not apply docs or skill changes without explicit approval.
- Choose the target by audience: future agents, human developers, or reusable cross-project behavior.

## Workflow

1. Read the workflow reference.
2. Gather repository and session context.
3. Apply the high bar to decide whether anything should be codified.
4. Read the output reference and respond in that format.
